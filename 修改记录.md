### 4. 地图右键拖动平移（CSS Transform） + 目击拖拽升级（SortableJS）

**地图拖动平移（`js/phase2.js` `_initMapPan()` + `_centerMap()` + `_applyPan()`）**

- `.map-wrapper` 桌面端 `overflow: hidden`，不使用原生滚动条
- 模块级变量 `_panX`/`_panY` 维护累积偏移量，`_applyPan()` 统一渲染
- `mousedown`（`e.button === 2`）→ `isDragging = true`；`document.mousemove` → `_panX += e.movementX` / `_panY += e.movementY` → `canvas.style.transform = translate()`
- `mouseup`（document）/ `mouseleave`（wrapper）→ 结束拖拽；`contextmenu` `preventDefault()` 屏蔽右键菜单
- `_centerMap()`：首次 render 时计算 `(wrapperW - canvasW) / 2` 作为初始偏移，地图默认居中显示
- **`css/styles.css`**：`.map-wrapper` 有 `cursor: grab`，`.panning` 时 `grabbing`

**目击记录拖拽排序（`js/phase2.js` `_renderSightingList()` + `index.html` + `css/styles.css`）**

- CDN 引入 SortableJS v1.15.6（`index.html`，在业务 JS 前加载）
- `Sortable.create(container, { animation: 150, handle: '.sighting-item-row', filter: '.sighting-action-btn', ghostClass/dragClass/chosenClass, onEnd })`
- `onEnd` 中调用 `State.reorderPath(oldIndex, newIndex)` 同步状态，然后 `render()` 重建列表
- `filter: '.sighting-action-btn'` 防止点击修改/删除按钮误触发拖拽；每次 `render()` 前 `destroy()` 旧实例
- 目击记录列表包含序号徽章、房间名、目击玩家、悬停显示的修改/删除按钮；修改复用浮层、删除调用 `State.removeFromPath()`
- **`js/state.js`**：新增 `reorderPath(fromIndex, toIndex)` 方法
- CSS：`.sighting-ghost`（半透明蓝色虚线占位）、`.sighting-drag`（拖拽中阴影高亮）、`.sighting-chosen`（选中态）

### 5. 目击输入重构：右上角数字面板

**HTML & CSS（`index.html` + `css/styles.css`）**
- 废弃 `#popover-input` 文本框方案，改为 `#popover-number-grid` 数字网格（CSS Grid `repeat(5, 1fr)`）
- `#sighting-popover` 放置在 `.map-wrapper` 内部，`position: absolute; top: 12px; right: 12px;`（相对于地图容器定位）
- `PLAYER_COLORS` 常量整体升级为高亮同色系（`js/data.js`）：#2 深蓝→亮蓝、#3 深绿→亮绿、#5 暗红→浅红、#8 深棕→琥珀、#9/#14 灰色提亮、#10/#13 暗紫/暗枚红→浅色 — 确保在深色面板背景下所有号码均可辨认
- `.popover-num-btn`：深色背景 `#2b2b2b`、`font-weight: 800`、`text-shadow: 0 1px 3px rgba(0,0,0,0.9)` — 与高亮 PLAYER_COLORS 配合达到极限清晰
- `.popover-num-btn.active`：白色边框 `2px solid #fff` + 微发光，不覆盖 JS 设置的文字颜色
- 底部操作栏「清空」+「保存」；移动端适配 4 列网格

**JS 逻辑（`js/phase2.js`）**
- `_openPopover()`：读取 `config.playerCount` 动态生成 1..N 号按钮，每按钮 `style.color = PLAYER_COLORS[n]` 赋予玩家专属颜色；已目击号码自动标记 `.active`
- 每个按钮绑定 `click` → `classList.toggle('active')` 切换选中/取消
- `_savePopover()`：遍历 `.popover-num-btn.active` 收集号码数组，升序排列后写入 `State.setSighting()` → `render()`
- 「清空」按钮：批量移除所有 `.active`；Escape 键关闭浮层

**地图节点圆点对比度（`css/styles.css`）**
- `.node-player-dot`：追加 `font-weight: 800` + `text-shadow: 0 1px 2px rgba(0,0,0,0.95)`，确保浅色编号在黑色圆点背景上清晰可辨
